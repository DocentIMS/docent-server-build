# Pre-Flight Checklist for Server Build

Gather everything in this checklist BEFORE starting `phase0-bootstrap.sh`.
The bootstrap script will prompt for these values one at a time, so having
them ready makes the build go smoothly.

All passwords are auto-generated by phase0 and saved to `secrets.local`.
You only need to provide the items in sections 5 and 7.

## 1. INFRASTRUCTURE

- [ ] Fresh Ubuntu VM (24.04 LTS recommended), with sudo access
- [ ] **Public IPv4 address** of the new VM: ______________
- [ ] **VPS provider portal access** (you'll need this to set reverse DNS / PTR
      after phase 4 generates the mail hostname)
- [ ] Confirm provider does NOT block outbound port 25 (required for sending mail)

## 10. POST-BUILD MANUAL STEPS

After all phases complete, you'll still need to:

### A. Add DNS records at your DNS provider:

| Record Type | Name | Value |
|---|---|---|
| A | `<domain>` | server IP |
| A | `www.<domain>` | server IP |
| A | `mail.<domain>` | server IP |
| MX | `<domain>` | `mail.<domain>` priority 10 |
| TXT (SPF) | `@` | `v=spf1 a:mail.<domain> ~all` |
| TXT (DKIM) | `default._domainkey` | (provided by phase 4 output) |
| TXT (DMARC) | `_dmarc` | `v=DMARC1; p=none; rua=mailto:<your-email>` |
| CAA (optional) | `@` | `0 issue "letsencrypt.org"` |

### B. Set reverse DNS (PTR) at VPS provider:

PTR for server IP → `mail.<domain>`

### C. WordPress setup:

Visit `https://<domain>/wp-admin/install.php` — set site title, admin
username (NOT `wayne` — separate WP-only admin account), admin password.

### D. Project Contacts:

Pre-populate the shared address book by importing vCard/CSV via the
Roundcube contacts UI, or by direct SQL insert with the dummy user
`_project_contacts_user_`.
