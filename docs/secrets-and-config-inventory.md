# Secrets and Configuration Inventory

Complete map of every configurable value, secret, and decision in the
docent-server-build phase scripts. This is the technical reference for
`phase0-bootstrap.sh` — every item here either becomes a phase0 prompt,
gets read from `secrets.local` or `tenant.local` at runtime, or stays
hardcoded as a system convention.

---

## Section D: Auto-Generated Secrets (saved to secrets.local)

Phase0 generates strong random values for each, then writes to `secrets.local`.
Each phase script reads from `secrets.local` at runtime instead of generating
its own (the existing `openssl rand` generation logic is kept as a fallback for
backward compatibility, but only fires if the variable is unset).

| Secret | Bytes/Format | Final destination on server | File mode |
|---|---|---|---|
| `ADMIN_PW` | base64, 22 chars | System password for $ADMIN_USER | n/a |
| `STAFF_PW` | base64, 22 chars | System password for $STAFF_USER | n/a |
| `ROOT_DB_PW` | base64, 28 chars | `/root/.my.cnf` | 0600 |
| `MAIL_DB_PW` | base64, 28 chars | `/etc/postfix/mysql-*.cf` | 0640 |
| `TEST_MAILBOX_PW` | base64, 22 chars | MariaDB virtual_users (hashed with `doveadm pw`) | n/a |
| `ROUNDCUBE_DB_PW` | base64, 28 chars | `/etc/roundcube/config.inc.php` | 0640 |
| `ROUNDCUBE_DES_KEY` | base64, 24 chars | `/etc/roundcube/config.inc.php` | 0640 |
| `WP_DB_PW` | base64, 28 chars | `/srv/www/<domain>/wp-config.php` | 0640 |

should NOT be exposed in phase0 because changing them breaks the install.

| Variable | Value | Why hardcoded |

| `VMAIL_UID` / `VMAIL_GID` | `5000` / `5000` | Reserved for vmail |
| `PLONE_HOME` | `/home/plone` | Plone convention |
| `PLONE_USER` | `plone` | Plone convention |
| `BACKUP_DIR` | `/var/backups/mysql` | Standard backup location |
| `BACKUP_RETENTION_DAYS` | `14` | Reasonable default |
| `ROOT_DEFAULTS_FILE` | `/root/.my.cnf` | MySQL convention |
| `ROUNDCUBE_DIR` | `/usr/share/roundcube` | Debian package install location |
| `ROUNDCUBE_URL_PATH` | `/mail` | Debian package default |
| `MAIL_DB` | `mailserver` | Internal naming |
| `MAIL_DB_USER` | `mailuser` | Service account, not human-facing |
| `ROUNDCUBE_DB` | `roundcube` | Internal naming |
| `ROUNDCUBE_DB_USER` | `roundcube` | Service account, not human-facing |

## Section K: secrets.local File Format

Phase0 generates this file at the repo root. It is `.gitignore`d so it is
never committed. Save it to your password manager after phase0 completes.

```bash
# ============================================================================
# secrets.local - Generated by phase0-bootstrap.sh on <date>
# ============================================================================
# THIS FILE CONTAINS PASSWORDS. KEEP IT SECURE.
# Listed in .gitignore. Save a copy to your password manager.
# Each phase script reads this at runtime.
# ============================================================================

# ---- USER-PROVIDED SECRETS ----
RC_PLUS_LICENSE_KEY="RCP-..."
XAI_API_KEY=""                   # blank if AI not used

# ---- AUTO-GENERATED SECRETS ----
ADMIN_PW="..."
STAFF_PW="..."
ROOT_DB_PW="..."
MAIL_DB_PW="..."
TEST_MAILBOX_PW="..."
ROUNDCUBE_DB_PW="..."
ROUNDCUBE_DES_KEY="..."
WP_DB_PW="..."
```

## Section L: tenant.local File Format

Phase0 also generates a non-secret tenant config file at the repo root.
This one is also `.gitignore`d (per-tenant config, not committed).

```bash
# ============================================================================
# tenant.local - Generated by phase0-bootstrap.sh on <date>
# ============================================================================
# Non-secret per-tenant configuration. Read by all phase scripts.
# ============================================================================

PRIMARY_DOMAIN="acmemuseum.com"

SSH_PORT="2222"

```
